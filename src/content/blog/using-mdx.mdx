MDX is useful when a normal markdown post needs one tiny interactive or reusable UI block.

## Why I use it for Linux notes

I keep most posts in plain markdown, but MDX is great for repeated checklists.

<ul>
	<li>Confirm backups are healthy.</li>
	<li>Run package updates with release notes open.</li>
	<li>Reboot only after checking running services.</li>
</ul>

## Keep MDX boring

Use MDX for small reusable fragments, not whole app logic. The cleaner your content files stay, the easier they are to maintain.
